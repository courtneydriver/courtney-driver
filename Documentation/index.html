1.2 SASS / CSS files

All compiled CSS assets are located in the assets/css directory:

  • libs.css - all styles from libraries
  • app.css - main styles

All precompiled SASS files are located in the resources/sass directory:

  • libs.scss - entry point for libs.css styles
  • app.scss - entry point for app.css styles
  • _variables.scss - SASS variables
  • _mixins.scss - SASS mixins
  • patials - SASS partials directory
    • _animations.scss - animations
    • _basic.scss - basic styles
    • _form.scss - contact form styles
    • _navigation.scss - navigation styles
    • _preloaders.scss - styles of all preloaders
    • _sections.scss - styles of each theme section (Blog, Portfolio etc.)

1.3 JavaScript files and Vue.js modules

All compiled JavaScript assets are located in the assets/js directory:

  • libs.js - compiled JS libraries
  • app.js - compiled template specific scripts with dependencies
  • particles.min.js - particles module

All precompiled JavaScript files are located in the resources/js directory:

  • libs.js - here you can require libraries
  • app.js - entry point for template specific scripts
  • bootstrap.js - importing and defining
  • Event.js - Event JS class
  • components - Vue.js components directory
    • ContactForm.vue - Contact form module
    • FormInput.vue - Form input module
    • Counter.vue - Counter module
    • IntroTyping.vue - Intro Typing module
    • Portfolio.vue - Portfolio module
    • PortfolioItem.vue - Portfolio Item module
    • Skill.vue - Skill module
    • StartArrow.vue - Start Arrow module
    • Testimonials.vue - Testimonials module
    • Testimonial.vue - Testimonial item module
  • directives - Vue.js components directives directory
    • Carousel.js - OWL Carousel directive
    • MagnificPopup.vue - Magnific Popup directive
  • scripts - other scripts directory
    • navigation.js - navigation scripts

1.4 Settings files

  • settings/config.js - JavaScript settings
  • settings/config.php - PHP settings
  • settings/particles-config.json - Particles module settings
  • webpack.config.js - Webpack configuration file

1.5 Other files

  • assets/images - all images of the template

3.2 Settings

All basic settings are located in settings directory:

  • settings/config.js - JavaScript settings
  • settings/config.php - PHP settings
  • settings/particles-config.json - Particles module configuration

Config.js

Here are located JavaScript specific settings like modules configurations, translations etc. Each part of the file was descripted in comments, for example:

/*
|--------------------------------------------------------------------------
| Testimonials slides configuration
|--------------------------------------------------------------------------
| Tetimonial module uses OWL Carousel plugin. Here you can configure all
| its behavior. All available options you can find on the
| https://owlcarousel2.github.io/OwlCarousel2/docs/api-options.html
*/

testimonials:
{
  items: 1, // The number of items you want to see on the screen
  autoplay: true, // Autoplay
  autoplaySpeed: 1000, // Autoplay speed
  loop: true // Infinity loop. Duplicate last and first items to get loop illusion
},

Config.php

The template uses PHP to send emails and input validation. Here you can configure all of it. Bellow you can see a little part of the file:

'contact_form' => [
  // Basic settings
  'to' => 'your@email.com',
  'defaultSubject' => 'Message from Contact Form',

  // Validation rules
  'validation' => [
    'name'    =>  'required',
    'email'   =>  'required|email',
    'message' =>  'required|min:20',
  ],

  // Validation messages
  'validationMessages' => [
    'required'  => 'This field is required',
    'email'   => 'Given email address is incorrect',
    'min:20'  => 'Enter more than 20 characters'
  ],

Particles-config.json

The file contains configuration of Particles.js module. If you are not familiar with that, you can get a custom configuration by using Online Generator from this website.

3.4 Animate On Scroll

The template uses AOS - Animate On Scroll library. In order to configure global settings go to config.js file.

/*
|--------------------------------------------------------------------------
| AOS - Animate on Scroll
|--------------------------------------------------------------------------
| This template uses AOS - small library to animate elements on your page 
| when you scroll. Here you can configure global settings.
| All available options you can find at https://github.com/michalsnik/aos
*/
  AOS:
  {
    offset: 120, // Change offset to trigger animations sooner or later (px)
    duration: 600, // Duration of animation (accepted values from 50 to 3000, with step 50ms)
    easing: 'ease-out-quart', // Choose timing function to ease elements in different ways 
    delay: 100, // Delay animation (ms)
    anchor: null, // Anchor element, whose offset will be counted to trigger animation instead of actual elements offset
    placement: 'bottom-top', // Anchor placement - which one position of element on the screen should trigger animation
    once: false, // Choose wheter animation should fire once, or every time you scroll up/down to element      
},

Customization

Method 1: Precompiled source files (for advanced)

Precompiled source files of each component you can find resources/js/components directory. After every modification, there is a need to run compilation by using one of following commands: npm run dev, npm run build or npm run production. More information about it you can find in Webbapck section.

Method 2: Replacing original templates in HTML file by using inline-template syntax

If you are not familiar with Webpack and npm, you can modify each of Vue.js component by using inline-template syntax. In this case there is no need to compile source files.

When the inline-template special attribute is present on a child component, the component will use its inner content as its template, rather than treating it as distributed content.

In order to modify component template, just follow the steps bellow:

  1. Given we have standard Vue.js component 
    <skill percent="80">HTML - <span class="c-silver">5 years of experience</span></skill>
  2. Put inline-template attribute into component tag 
    <skill percent="80" inline-template>HTML - <span class="c-silver">5 years of experience</span></skill>
  3. Grab template specific code from component file (eg. resources/js/components/Skill.vue) and put it into component tag 
    <skill percent="80" inline-template>
  <div class="Skill">
    <div class="Skill__head">
      <span class="Skill__title flex">HTML - <span class="c-silver">5 years of experience</span></span>
      <span class="Skill__percent">{{percent}}%</span>
    </div>
    <div class="Skill__bar">
      <div class="Skill__bar-filling"></div>
    </div>
  </div>
</skill>
  4. Now you are able to modify HTML code directly in HTML file.

4. Webpack (for advanced)

The template is distributed with configured webpack.config.js and package.json file. So you already have the ready environment for working with SASS, JS (ES2015) and Vue Components templates.

4.1 Resource Files

All precompiled files are located in the resources folder. Below are listed the most important directories:

  • resources/sass - SASS (SCSS) files
  • resources/js - JavaScript files
  • resources/js/components - Vue Componetns specific files (each of component has own file with template and JS)
  • resources/images - images used by SASS and compiled by Webpack Image Loader

4.2 Setup

  1. Install Node.js according the instruction.
  2. By using console (command prompt) install all project dependencies. 
    npm install
  3. And that's it! You are ready to go.

In order to modify Webpack settings go to webpack.config.js file.

4.3 Commands

  • npm run build - run a single compilation of resources files.
  • npm run dev - run a compilation of resources files with watch option.
  • npm run production - run a compilation of resources files with minimization (uglify).
